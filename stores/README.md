# Stores

Domain-per-store **purchase lists**: one file you open while you are physically (or virtually) at that kind of store. Tools and ergonomics live with cooking in [`../recipes/tools.md`](../recipes/tools.md); recipe procedures in [`../recipes/README.md`](../recipes/README.md).

**Upstream:** [Notes root](../README.md)

## Index

| File | Purpose |
| --- | --- |
| **[pharmacy.md](pharmacy.md)** | Vitamins, emergency meds, personal care, skin — what to grab when you walk into a pharmacy. |
| **[supermarket.md](supermarket.md)** | Cleaning, hygiene, grocery (perishable + non-perishable) and the **weekly batch** coverage table for recipes. |
| **[ecommerce.md](ecommerce.md)** | Online-only buys (electronics accessories, durables, niche items) — placeholder structure. |

## Conventions

- **Core items** = always keep stocked.
- **Batch items** = buy when cooking for multiple days.
- Substitutions are fine; the goal is one file to scan per store, not a strict SKU list.
