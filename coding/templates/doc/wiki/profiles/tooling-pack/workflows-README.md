# Workflows

## Verify (local health)

- Install dependencies per root **`README.md`**
- Run **format**, **lint**, **test** (or your repo’s umbrella command)
- Fix failures before opening a PR

## Publish (share changes)

- Merge or rebase per team **`main`** policy
- **Push** branch
- Open or update **PR** with title/body templates from this pack’s **`pr/`** and **`shared/`**

## Issue → branch → PR (linear habit)

1. Confirm issue / ticket scope
2. Create branch (see [`../../../../git/start-branch-naming.md`](../../../../git/start-branch-naming.md))
3. Implement and verify
4. Push and open PR
