# Skeleton: doc page explanation

**Conventions:** [`markdown-and-docs-conventions.md`](markdown-and-docs-conventions.md)

Narrative body for a **`docs/`** page that documents **one** subsystem, tool, or convention folder (e.g. `packages/auth/`, `tooling/ci/`).

---

## Fill-in pattern

1. **What it does** — one short paragraph (outcome for readers).
2. **When to use** — scenarios or commands.
3. **Boundaries** — **Do** / **Do not** bullets.
4. **Related** — links to sibling docs or runbooks in the **same** repository.

---

## Skeleton

```markdown
## Overview

<Prose: outcome and behavior.>

## When to use

- …

## Do and do not

- **Do:** …
- **Do not:** …

## Related

- [`../other-topic/README.md`](../other-topic/README.md) — …
```
