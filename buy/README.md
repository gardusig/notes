# 📌 Stores

Domain-per-store **purchase lists**: one file you open while you are physically (or virtually) at that kind of store. Tools and ergonomics live with kitchen notes in [`../cook/tools.md`](../cook/tools.md); procedures live in [`../cook/README.md`](../cook/README.md).

**Upstream:** [Notes root](../README.md)

## 📑 Index

| File | Purpose |
| --- | --- |
| **[pharmacy.md](pharmacy.md)** | Supplements, common-aisle relief, personal care, skin — category checklist for a pharmacy run. |
| **[supermarket.md](supermarket.md)** | **Perishable** produce, meat, eggs, frozen fruit — the fresh run — plus **weekly batch (perishable)**. Pantry batch: [`ecommerce.md`](ecommerce.md#pantry-weekly-batch). |
| **[ecommerce.md](ecommerce.md)** | **Online supermarket** (most food, dry grocery, pantry weekly batch) and **standard purchases** (electronics, durables, cleaning, hygiene, niche items). |

## 📜 Conventions

- **Core items** = always keep stocked.
- **Batch items** = buy when cooking for multiple days.
- Substitutions are fine; the goal is one file to scan per store, not a strict SKU list.
