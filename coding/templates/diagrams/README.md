# Diagram templates

**Canonical scaffolds** live in this directory (`templates/diagrams/`). Use them for mermaid in docs, PR bodies, and GitHub issues. For **color conventions** in class diagrams and similar, start from [`palette.md`](palette.md).

## Index

1. [`palette.md`](palette.md) — `classDef` ramps for colored mermaid.
2. [`examples.md`](examples.md) — sample figures.
3. [`template-dag.md`](template-dag.md) — fan-out / delegation.
4. [`template-decision.md`](template-decision.md) — decision tree.
5. [`template-deployment-topology.md`](template-deployment-topology.md) — environments + logical deploy zones (full stack / study).
6. [`template-er.md`](template-er.md) — `erDiagram` for schema drafts and study.
7. [`template-flowchart.md`](template-flowchart.md) — top-down pipeline.
8. [`template-layers.md`](template-layers.md) — layered or split-service architecture.
9. [`template-mindmap-study.md`](template-mindmap-study.md) — topic mind map (study).
10. [`template-request-path.md`](template-request-path.md) — browser → API → DB trace.
11. [`template-sequence.md`](template-sequence.md) — sequence diagram.
12. [`template-stakeholder-flow.md`](template-stakeholder-flow.md) — stakeholder lanes.
13. [`template-state.md`](template-state.md) — state diagram.
14. [`template-user-journey.md`](template-user-journey.md) — user journey.
